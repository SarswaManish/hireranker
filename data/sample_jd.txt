Senior Backend Engineer
FinTrack — San Francisco, CA (Remote-friendly)

About FinTrack
--------------
FinTrack is a Series B fintech startup building the next generation of spend
management and financial automation tools for mid-market businesses. Our
platform processes over $2B in transactions annually and serves 1,200+
customers across North America. We are a team of 85, growing fast, and backed
by Sequoia Capital and Andreessen Horowitz.

Role Overview
-------------
We are looking for a Senior Backend Engineer to join our Core Platform team.
You will design and build the APIs, data pipelines, and microservices that
power FinTrack's transaction processing, reconciliation, and reporting
capabilities. You will work closely with product managers, frontend engineers,
and data scientists to ship reliable, scalable software in a regulated
financial environment.

This role requires someone who writes clean, well-tested code, cares deeply
about system reliability, and can navigate the complexity of financial data
with precision.

What You'll Do
--------------
- Design, build, and maintain high-throughput REST and event-driven APIs
  serving millions of financial transactions per day.
- Own and evolve our core transaction processing service (Python/Django,
  PostgreSQL, Kafka).
- Lead backend architecture discussions and contribute to technical roadmap
  planning.
- Collaborate with the data engineering team to build real-time reconciliation
  and reporting pipelines using dbt and Spark.
- Ensure all systems meet PCI-DSS compliance requirements and pass SOC 2 Type
  II audits.
- Mentor junior and mid-level engineers through code reviews and pair
  programming.
- Participate in on-call rotation for production incidents (PagerDuty, roughly
  1 week per quarter).
- Contribute to internal developer tooling and CI/CD improvements.

Requirements
------------
Must-have:
- 5+ years of professional software engineering experience.
- 3+ years building production backend systems with Python (Django or FastAPI).
- Deep expertise with PostgreSQL — query optimization, indexing strategies,
  migrations, and schema design for financial data.
- Experience with asynchronous task processing (Celery, RQ, or similar).
- Solid understanding of REST API design principles and HTTP semantics.
- Experience with Docker and Kubernetes in a production environment.
- Strong proficiency in Git and collaborative development workflows.
- Experience writing comprehensive unit and integration tests (pytest).
- Ability to communicate clearly in writing and in cross-functional meetings.

Nice-to-have:
- Experience with event streaming platforms (Apache Kafka, AWS Kinesis, or
  similar).
- Familiarity with financial industry standards: ISO 20022, ACH/wire
  processing, PCI-DSS, SOC 2.
- Experience with AWS services (RDS, SQS, S3, Lambda, ECS/EKS).
- Knowledge of data warehousing and analytics pipelines (dbt, Snowflake,
  Redshift).
- Prior experience at a fintech, payments company, or bank.
- Open source contributions or public GitHub portfolio.
- Experience with Redis for caching and distributed locking.
- Understanding of cryptography fundamentals as applied to secure API design.

Tech Stack
----------
- Backend:   Python 3.11, Django 4.2, Django REST Framework, Celery
- Databases: PostgreSQL 15, Redis 7
- Messaging: Apache Kafka, AWS SQS
- Cloud:     AWS (EKS, RDS Aurora, S3, CloudFront, Secrets Manager)
- CI/CD:     GitHub Actions, ArgoCD
- Monitoring:Datadog, PagerDuty, Sentry
